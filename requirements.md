# 需求文档（requirements.md）

## 1. 项目概述

### 1.1 项目名称
英语学习助手（Chrome 扩展）/ Word Master

### 1.2 背景与愿景
用户在日常浏览英文文字站点与视频网站（如 YouTube / Bilibili / Netflix 等）时，会频繁遇到不认识的单词或表达。传统方式（复制、打开词典 App、切换窗口）打断沉浸式阅读/观看，导致学习成本高、难以坚持。

本项目的愿景是：**让“遇到生词→立即查询→当场建立有趣的第一次记忆→后续愉悦复习”成为低阻力、可持续的习惯**，最终让英语成为用户获取信息的主要语言来源，word master不止于查询单词，更让您轻松记住单词

### 1.3 核心方法论（产品原则）
- **即时性**：查词动作应当在当前页面内完成，尽量避免跳转与上下文切换。
- **有趣的首次编码**：允许用户为每个单词添加“符合自己”的联想记忆（谐音、字形、意义联想、词根词缀等），把枯燥的首次记忆变得有趣。
- **可积累的复习线索**：在查词卡片里直接呈现“我的联想记忆”和“复习次数”，让每次复习更快、更愉悦。
- **基于遗忘的反馈**：复习次数越多意味着忘得越多，应突出提示需要重点记忆的词。
- **本地优先与隐私优先**：用户的自定义联想记忆与学习数据默认保存在本地（本机 SQLite/浏览器本地缓存），不强制上云。

### 1.4 产品形态与组成
系统由两部分组成：
- **Chrome 扩展（Manifest V3）**：负责网页内触发查词、显示单词卡片、录入/展示联想记忆、记录复习、弹窗手动查词与翻译。
- **本地 API 服务（Flask + SQLite）**：负责词典查询整合、数据持久化、复习次数与间隔复习计算、数据导出与统计接口。

> 关键约束：扩展在“完整模式”下依赖本地服务 `http://127.0.0.1:5000`；在服务不可用时，扩展会降级为“在线词典 + 本地缓存”的有限能力。

## 2. 用户画像（Personas）

### 2.1 主要用户（核心目标人群）
- **沉浸式输入型学习者**
  - 场景：经常看英文网站/论坛/文档、观看英文视频并依赖字幕理解内容。
  - 目标：通过真实语境积累词汇与表达，使英语成为主要信息来源。
  - 痛点：频繁查词打断沉浸；记忆效率低；复习枯燥、难坚持。

- **自我驱动的词汇记忆优化者**
  - 场景：已使用墨墨背单词等工具，但希望把“遇到的词”与“自己的联想记忆”沉淀到可复用的库中。
  - 目标：建立个性化联想记忆库，提升复习愉悦感与记忆牢固度。
  - 痛点：现有软件联想记忆不个性化；记录分散；难以在“遇到词的当下”快速记录。

### 2.2 次要用户（边缘人群）
- **英文资料工作者（开发/科研/运营）**：需要高频阅读英文资料，追求“查词不打断”。
- **备考/应试用户**：希望把遇到的词快速入库并统计复习次数（但本产品不以刷题/题库为核心）。

### 2.3 不适用人群（不重点服务）
- 需要系统化课程、语法教学、口语对话练习的用户（本项目不覆盖课程体系与口语陪练）。
- 对隐私/本地服务部署完全不愿意配置的用户（现阶段完整能力依赖本地服务）。

## 3. 核心痛点与价值主张

### 3.1 核心痛点
- **查词打断沉浸**：复制-切应用-粘贴-返回的成本高，导致不愿查或查完即忘。
- **首次记忆枯燥**：只看到释义，缺少“有趣的首次编码”，后续复习容易痛苦。
- **联想记忆高度个性化**：同一单词对不同人最有效的联想方式不同，通用助记价值有限。
- **复习缺乏反馈**：不知道哪些词总忘、哪些词已掌握，难以分配注意力。
- **数据分散/不可积累**：查词记录、助记内容、复习次数散落在多处，难以沉淀为长期资产。

### 3.2 价值主张
- **一键/双击即查**：在原页面内完成查词与复习。
- **用户自定义联想记忆**：把“适合自己”的联想直接沉淀到单词卡片里，形成可复用线索。
- **快乐复习**：每次遇到同一单词，都能看到自己写过的联想记忆，降低复习痛苦、提高坚持率。
- **复习次数可视化**：用“忘得多→需要重点”的朴素反馈机制引导学习策略。

## 4. MVP 范围（必须包含）

### 4.1 触发与入口（扩展端）
- **网页内双击选中文本触发查词**
  - 双击选中长度合理（例如 2–50 字符）的英文单词/短语后，弹出单词卡片。
- **快捷键触发查词**
  - 在页面中选中文本后，使用快捷键（当前实现为 `Ctrl + Q`）触发查词。
- **扩展弹窗（Popup）手动输入查词**
  - 点击扩展图标打开弹窗，输入单词并查询。

### 4.2 单词卡片（网页内展示）
- **展示基础信息**
  - 单词、音标（若可得）、常见释义（默认展示前 3 条）。
- **展示联想记忆（自定义助记）**
  - 展示该单词已保存的自定义联想记忆列表（含可选解释）。
- **展示学习统计**
  - 展示复习次数 `review_count`（若可得）。
- **卡片内关键操作**
  - 添加联想记忆（输入助记文本 + 可选解释 + 可选“私有”标记）
  - 记录复习（为该单词复习次数 +1，更新下次复习时间与掌握度）

### 4.3 数据存储（服务端 + 本地）
- **服务端 SQLite 持久化**
  - 存储单词信息（释义/音标/例句等）、自定义助记、复习次数与复习计划字段（next_review_date / master_level）。
- **扩展端本地缓存（localStorage）**
  - 单词查询结果（含自定义助记、复习次数等字段）以 `word_data_<word>` 键缓存，加快二次打开速度。
  - 当服务端更新（新增助记、记录复习）后，扩展应清理该单词缓存并重新拉取最新数据。

### 4.4 词典来源与降级策略
- **优先本地词典（ECDICT）**：若 `ecdict.db` 可用，优先返回本地查询结果。
- **其次在线免费词典 API**：使用 `dictionaryapi.dev` 获取英文释义与例句，服务端会尽力将释义/例句翻译为中文（若翻译失败则保留英文）。
- **服务不可用时（扩展端降级）**
  - 若本地服务不可用：扩展应提示“服务器未连接”，并允许用户重试连接。
  - 若本地服务返回错误：扩展可使用在线词典 API 直接查询并展示（但助记/复习等需要服务端的能力将受限或不可用）。

### 4.5 弹窗（Popup）文本翻译
- **文本翻译**
  - 在弹窗中输入一段文本，选择源/目标语言，调用翻译服务获得翻译结果并展示。

## 5. 关键用户故事（User Stories）

### 5.1 查词与沉淀
- 作为一个在网页阅读英文内容的人，我希望**双击一个生词就能看到释义**，这样我不需要切换应用，阅读不被打断。
- 作为一个在看视频字幕的人，我希望能**快速查字幕里选中的词**，这样我可以边看边理解。
- 作为一个自我驱动学习者，我希望**对每个词写下自己的联想记忆**（谐音/字形/词根/意义联想等），这样第一次记忆更有趣也更牢固。

### 5.2 快乐复习与反馈
- 作为一个会反复忘词的人，我希望每次查词都能看到**我以前写过的联想记忆**，这样复习变得轻松且情绪更好。
- 作为一个需要分配注意力的人，我希望看到**复习次数**，次数越高我越能意识到它是“需要重点”的词。

### 5.3 可持续与可迁移
- 作为一个担心数据丢失的人，我希望学习数据能**保存在本地并可导出**，这样我能迁移到其他设备或做备份。

## 6. 功能清单（MVP 细化）

### 6.1 Chrome 扩展（网页侧）
- **查词触发**
  - 双击选中触发
  - 选中后快捷键触发
  - 弹窗手动输入触发
- **展示单词卡片**
  - 单词、音标、常见释义（前 3 条）
  - 自定义联想记忆列表（含解释）
  - 学习统计（复习次数）
- **联想记忆管理**
  - 新增联想记忆（文本 + 解释 + 私有标记）
  - 删除联想记忆（当前实现存在“删除”入口）
  -（可选/若已实现）使用某条联想记忆（记录“常用/喜欢”的行为）
- **复习记录**
  - 点击“记录复习”后写入服务端并刷新卡片数据
- **翻译**
  - 单词/文本翻译（弹窗内文本翻译；网页卡片内可选翻译按钮）

### 6.2 本地 API 服务（服务端）
- **查词接口**
  - `GET /api/word/<word>`：返回 WordInfo（释义、例句、自定义助记、复习次数等）
- **新增单词接口**
  - `POST /api/word`：添加新单词到数据库（可选携带释义/音标/例句）
- **助记接口**
  - `POST /api/mnemonic`：为单词新增自定义联想记忆
- **复习接口**
  - `POST /api/review`：记录复习结果（难度 1–5），更新复习次数/掌握度/下次复习时间
- **统计/导出**
  - `GET /api/statistics`：返回总词数、掌握词数、待复习词数、自定义助记数、总复习次数等
  - `GET /api/export?format=json`：导出 JSON 数据文件

## 7. 非功能需求（NFR）

### 7.1 性能
- **查词响应时间**
  - 有缓存：卡片展示 < 200ms（允许先展示缓存，再异步刷新服务端结果）
  - 无缓存：常规情况下 < 2s；最差情况下（在线 API 慢/翻译失败）可提示加载中或降级展示。

### 7.2 可用性与可靠性
- 本地服务不可用时，扩展应明确提示并提供“重试连接”入口。
- 服务器应避免因多进程导致 SQLite 锁死（当前建议关闭 debug 多进程）。

### 7.3 隐私与安全
- **默认本地存储**：自定义助记与学习数据默认只保存在本机（SQLite + 浏览器 localStorage 缓存）。
- **最小化采集**：不采集用户的网页内容全文；只对用户主动选中的词/短语进行查询与存储。
- **可导出**：允许用户导出学习数据以自备份/迁移（MVP 已包含接口）。
- **网络访问说明**：在线词典与翻译会对外发起请求，应在文档/设置中告知访问的域名与用途。

### 7.4 兼容性
- Chrome 扩展：Manifest V3。
- 目标网页：通用网页 + 特定视频网站（YouTube/Bilibili/Netflix/TED 等）。

## 8. 数据与领域模型（概念）

### 8.1 单词（Word）
字段（概念层）：
- `word`：单词本体（小写）
- `phonetic`：音标（可空）
- `definitions`：释义列表（词性 `pos` + 释义 `def`）
- `examples`：例句列表（`english` + `chinese`，数量可控）
- `review_count`：复习次数
- `master_level`：掌握度（0–1）
- `next_review_date`：下次复习时间

### 8.2 自定义联想记忆（CustomMnemonic）
- `text`：联想记忆内容（用户输入）
- `explanation`：解释（可选）
- `rating`：评分/偏好（可选）
- `is_private`：是否私有（为未来“共享助记库”预留）

---

## 9. 交互流程（MVP）

### 9.1 网页内双击查词
1. 用户在网页中双击选中单词/短语
2. 扩展在页面中央弹出单词卡片
3. 先读本地缓存并快速展示（若有）
4. 并行请求本地 API 服务获取最新信息
5. 用户可：
   - 点击“➕ 添加联想记忆”并提交
   - 点击“📊 记录复习”累加复习次数
6. 提交成功后卡片刷新，展示最新联想记忆与复习次数

### 9.2 点击扩展图标手动查词
1. 用户点击扩展图标打开弹窗
2. 输入单词并点击查询
3. 弹窗展示释义、联想记忆与复习次数
4. 可在弹窗内添加联想记忆、记录复习

## 10. 验收标准（Definition of Done / Acceptance Criteria）

### 10.1 基础可用（必须满足）
- **查词入口可用**
  - 在任意网页双击选中英文单词后能弹出卡片并展示查询结果。
  - 在弹窗输入单词点击“查询”能显示结果。
- **自定义联想记忆可用**
  - 在单词卡片中添加联想记忆提交成功后：
    - 再次查询同一单词能看到新增的联想记忆
    - 联想记忆内容与解释字段完整保存
- **复习记录可用**
  - 点击“记录复习”后：
    - 服务端 `review_count` 增加
    - 卡片/弹窗能显示最新 `review_count`
- **本地持久化可用**
  - 关闭浏览器/重启电脑/重启服务后，再次查询同一单词仍能看到历史联想记忆与复习次数。

### 10.2 降级与提示（必须满足）
- 本地服务未启动时：
  - 扩展能提示“服务器未连接”
  - 提供“检查连接/重试”入口
- 服务端查询失败时：
  - 扩展不崩溃，能给出明确错误提示；若可用在线词典则展示降级结果。

### 10.3 体验目标（建议达成）
- 二次查同一单词时优先从缓存展示，卡片能“秒开”（明显快于首次查询）。
- 卡片内容层级清晰：释义/联想记忆/统计/操作按钮一目了然。

## 11. 成功指标（可量化）

### 11.1 使用指标
- **日活/周活**：每周至少 3 天使用扩展查词。
- **查词次数**：每周查词 ≥ 50（可因用户差异调整）。
- **助记沉淀率**：被添加自定义联想记忆的单词占比（例如 ≥ 20%）。

### 11.2 学习效果指标（近似指标）
- **高复习次数词的下降趋势**：同一批高复习次数词在后续一段时间内复习频率下降。
- **用户主观体验**：复习是否“更愉悦/更愿意坚持”（可通过问卷/反馈收集）。

---

## 12. 边界与不做（MVP 不包含）
- 不做账号体系、云同步、多人共享助记库（`is_private` 仅为未来预留）。
- 不做系统化课程、语法教学、听说训练。
- 不做复杂的生词本管理 UI（标签/分组/筛选可作为后续迭代）。

---

## 13. 风险与依赖

### 13.1 依赖
- 本地服务（Flask）需运行在用户机器上并保持可达。
- 本地词典（ECDICT）可选：若缺失则依赖在线词典 API。
- 翻译服务/在线词典 API 的可用性会影响释义/例句的中文化质量。

### 13.2 风险
- 不同网站 DOM/字幕结构差异大，字幕提取/选择行为可能不稳定（需持续兼容）。
- 词典/翻译 API 的限流与网络不可达会导致体验下降（需做好降级与缓存）。

---

## 14. 后续迭代方向（Roadmap - 非 MVP）
- **重点词提示**：依据 `review_count` / `master_level` 在卡片上给出“重点记忆”提示与视觉强调。
- **复习队列 UI**：展示“今日待复习”列表（使用服务端 `next_review_date`）。
- **快捷添加语境**：把选中词所在句子/字幕片段作为上下文保存到 `learning_records`。
- **联想记忆模板**：为谐音/词根/字形等提供结构化输入模板，降低输入成本。
- **数据同步与备份策略**：一键备份、云同步（需明确隐私与安全方案）。
